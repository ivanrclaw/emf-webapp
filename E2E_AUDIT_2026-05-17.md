## 🔴 CRITICAL BUGS

### 1. Property Inspector no funciona para clases (nodes)
- **Descripción:** Al seleccionar un nodo (clase/enum/datatype) en el canvas o en el model tree, el Property Inspector sigue mostrando "Select an element to inspect its properties". Solo funciona cuando se selecciona una referencia.
- **Impacto:** Alto — no se pueden renombrar clases, cambiar abstract/interface, modificar super types, ni ver propiedades del clasificador desde el inspector.
- **Evidencia:** `document.querySelectorAll('[data-selected]')` → 0 elementos. No hay estado de selección en el DOM.
- **Reproducción:** Click en cualquier nodo del canvas → inspector no reacciona. Click en referencia → inspector muestra propiedades.

### 2. Export XMI falla por endpoint POST /metamodels/:id/export (500)
- **Descripción:** `POST /api/projects/:pid/metamodels/:mmid/export` con `{"format":"xmi"}` devuelve HTTP 500.
- **Workaround:** `GET /api/projects/:pid/xmi/:mmid/ecore` funciona correctamente.
- **Impacto:** Medio — el botón "Export .ecore" en la UI probablemente usa el endpoint que funciona (GET), pero si usa el POST, fallaría silenciosamente.
- **Causa probable:** Error en `serializeToXMI()` del `@emf-webapp/core` cuando se llama desde `EcoreTransformer.export()`.

## 🟡 MEDIUM ISSUES

### 3. Fechas en formato ISO crudo en "Recent Projects" (Landing)
- **Descripción:** En la landing page, las fechas se muestran como `2026-05-17T17:04:43.000Z`. En la vista de detalle del proyecto están formateadas como `5/17/2026`.
- **Impacto:** Bajo-medio — inconsistencia de UX, difícil lectura para humanos.
- **Fix:** Aplicar el mismo formatter de fechas que se usa en la vista de detalle.

### 4. Sin feedback visual al guardar (Save button)
- **Descripción:** El botón "Save" no muestra toast, notificación, ni cambio visual de confirmación.
- **Impacto:** Medio — el usuario no sabe si se guardó correctamente.
- **Fix:** Toast "Saved successfully" o indicador de estado.

### 5. Nombres genéricos sin edición inline
- **Descripción:** Las clases se crean como "NewClass1", "NewClass2" y los atributos como "newAttr" sin opción de renombrar inline en el canvas. Dado que el Property Inspector no funciona para clases (Bug #1), no hay forma de renombrar una clase desde la UI.
- **Impacto:** Alto para usabilidad — en Eclipse EMF/Sirius se puede renombrar con doble clic o F2.
- **Fix:** Habilitar edición inline en el nodo del canvas o arreglar el Property Inspector.

### 6. Onboarding tooltip tapa funcionalidad que describe
- **Descripción:** El tooltip "Explorer" cubre completamente el panel del explorador de archivos que intenta explicar.
- **Impacto:** Medio — contraproducente para nuevos usuarios.
- **Fix:** Reposicionar tooltip o añadir highlight/glow al elemento objetivo sin taparlo.

### 7. Explorer muestra proyectos antiguos sin limpiar
- **Descripción:** 13+ proyectos de pruebas en el Explorer (FixVerification, MTLFinalCheck, Debug, etc.).
- **Impacto:** Bajo — la DB no se limpió desde la última auditoría. No es un bug funcional.

### 8. Atributos/referencias se crean con nombres genéricos sin prompt
- **Descripción:** Al hacer clic en "+" para añadir atributo, se crea "newAttr: EString" directamente sin modal para nombrarlo. Lo mismo con referencias ("newRef").
- **Impacto:** Medio — requiere edición posterior (que no funciona para clases por Bug #1, pero para atributos podría haber inline edit).

## 🟢 MINOR / NICE-TO-HAVE

### 9. Botón "Validate" sin tooltip
- El botón Validate en la toolbar no tiene tooltip explicativo ni estado visual (loading, success).

### 10. Sin atajos de teclado
- No hay keyboard shortcuts para acciones comunes (Ctrl+S save, Delete eliminar, F2 renombrar).

### 11. Canvas sin grid visible
- El canvas no muestra grid de fondo — en herramientas como Sirius/Lucidchart el grid ayuda a alinear.

### 12. Navegación desde Explorer a vista de proyecto
- Hacer clic en un proyecto en el Explorer cuando ya estás en una tab de editor no abre la vista del proyecto. Requiere ir a Welcome → clic en proyecto.

### 13. Duplicados en Explorer
- "E2E Test Project" aparece dos veces en el árbol del Explorer.

**Veredicto:** La plataforma está en un **estado funcional sólido** para ser un drop-in replacement de EMF. El pipeline completo (crear proyecto → metamodelo → clases → referencias → código → XMI) funciona. Los bugs críticos (Property Inspector, Export POST) son corregibles y no bloquean el flujo principal gracias a workarounds existentes.

**Para ser un verdadero drop-in replacement de Eclipse EMF**, las prioridades serían:
1. **Arreglar Property Inspector para clases** — sin esto no se pueden renombrar clases ni editar propiedades fundamentales
2. **Edición inline de nombres** — doble clic para renombrar como en Eclipse
3. **Feedback visual de Save** — confianza del usuario
4. **Formatear fechas** — pulido profesional
