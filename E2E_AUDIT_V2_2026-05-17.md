## 🐛 New Issues Found

### Critical (🔴)

| # | Issue | Impact | Details |
|---|-------|--------|---------|
| C1 | **No M1 model XMI export endpoint** | High | `GET /api/projects/:pid/metamodels/:mmid/models/:modelId/export?format=xmi` returns 404. The UI has an XMI export button but no backend endpoint exists for model instance export (only metamodel export works). |

### Medium (🟡)

| # | Issue | Impact | Details |
|---|-------|--------|---------|
| M1 | **Onboarding tour not persisting** | Medium | `localStorage.setItem('emf-onboarding-done', 'true')` appears to not persist across page navigations. Onboarding popup reappears on every new page load/tab open. |
| M2 | **Duplicate `substring` case in MTLExecutor** | Medium | `packages/core/src/mtl/MTLExecutor.ts` line 444-445: `case 'substring':` appears twice. Second case duplicates first. Vite warning during test run. |
| M3 | **Edge mapping "Add" not visually reflected** | Medium | Clicking "Add edge mapping for NewClass1.newRef" in Spec tab doesn't visibly add the mapping to the Edges section. The section stays empty with just "Add Tool" button. |

### Minor (🟢)

| # | Issue | Impact | Details |
|---|-------|--------|---------|
| m1 | **Auto-generated name "New NewClass1"** | Low | When creating a model instance from the spec palette, the node is named "New NewClass1" (redundant prefix). Should be just "NewClass1" or prompt for custom name. |
| m2 | **Toast save feedback invisible** | Low | Save toast logic is correct but duration (3s) is too short to capture. CSS variables `--toast-bg`, `--toast-text` may not be defined in all themes. |
| m3 | **Monaco Editor not automation-friendly** | Low | Monaco uses a hidden textarea + contenteditable overlay, making it inaccessible to `browser_type`. Requires direct Monaco API (`monaco.editor.getModels()[0].setValue()`) which isn't available in the browser global scope. |
| m4 | **Backend tests not runnable** | Low | `packages/backend/tests/` has `e2e-xmi.test.ts` and `api.test.ts` but vitest is not installed in the backend package. `pnpm test` fails with "vitest: not found". |
